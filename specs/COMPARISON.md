## 根因

`create-spec.py` 只做了**代码提取**（API 路由 + Zod schema + 页面组件），没做**行为描述**。后者需要：
1. 读 Issue 中的需求描述
2. 读 E2E 测试中的 Given-When-Then
3. 理解业务逻辑后手写

## 修正方案

**不修改全部 26 个 spec**（太多且无差异内容都是模板），而是：

1. **修正 TEMPLATE.md** — 补上缺失的章节（Problem Statement / User Stories / Non-goals / Open Questions）
2. **修正 create-spec.py** — 生成时不再保留模板占位符，只填能确定的（API + Schema + 页面），其余留空等人工补齐
3. **按优先级补全** — 先从有 open Issue 的 3 个模块开始（relations #729 / timeline #730 / audit #733），试点后推广
