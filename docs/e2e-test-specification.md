## 二、用例四态覆盖说明

每条用例严格按照以下四种状态全覆盖：

| 状态 | 说明 | 覆盖率目标 |
|------|------|:----------:|
| **正常** | 标准操作路径，预期成功 | 100% 模块 |
| **异常** | API 失败 / 网络异常 / 非法输入 | 100% 模块 |
| **空** | 空数据 / 初始状态 / 全部删除 | 100% 模块 |
| **边界** | 长文本 / 特殊字符 / 极限值 | 关键业务模块 |

## 三、模块 1: 会话管理 (10 用例)

| 用例 | 标题 | 四态 | 描述 |
|:----:|------|:----:|------|
| S1 | 创建新会话 | 正常 | Given 用户打开书籍 → When 点击"新建会话" → Then 新建会话创建成功，输入框获得焦点 |
| S2 | 会话输入和发送 | 正常 | Given 有活动会话 → When 输入内容并回车 → Then 消息显示在对话区域，输入框清空 |
| S3 | 空会话状态 | 空 | Given 用户打开一个从未发送过消息的会话 → Then 显示空状态提示"开始和 AI 助手对话" |
| S4 | 会话列表切换 | 正常 | Given 有多个会话 → When 点击不同会话 → Then 对话内容切换到对应会话的历史记录 |
| S5 | 输入框特殊字符 | 边界 | Given 活动会话 → When 输入包含 HTML 标签和 emoji 的长文本 → Then 正确显示，无 XSS 问题 |
| S6 | 连续快速发送 | 边界 | Given 活动会话 → When 快速连续发送 5 条消息 → Then 所有消息按顺序显示，无丢帧 |
| S7 | 会话删除 | 正常 | Given 有存在的会话 → When 删除该会话 → Then 会话从列表中移除，对话区域提示"无活动会话" |
| S8 | 会话删除取消 | 正常 | Given 点击删除某会话 → When 在确认弹框中点击取消 → Then 会话仍存在，未被删除 |
| S9 | 会话重命名 | 正常 | Given 有存在的会话 → When 重命名为新名称 → Then 会话列表更新为新的名称 |
| S10 | API 失败错误处理 | 异常 | Given 将会话 API 返回 500 → When 加载会话列表 → Then 显示错误提示和重试按钮 |

## 十五、运行策略

### 环境变量

| 变量 | 默认值 | 说明 |
|------|:------:|------|
| `INKOS_E2E_REAL_LLM` | (空) | 设为 `1` 使用真实 LLM API |
| `INKOS_AGENT_LLM_STUB` | `1` | Agent 创作测试 LLM Mock 开关 |

### CI 配置

- 默认使用 `INKOS_AGENT_LLM_STUB=1` (Mock 模式)
- 不触发真实 LLM 调用
- 每个 PR 合并前必须全量 E2E 通过
