# InkOS Security Policy

**Last updated**: 2026-07-01

## Overview

InkOS prioritizes user data security. As a local-first AI writing tool, we are committed to software security and transparency.

## Data Handling & Storage

- **All data stored locally**: Creative content, configuration, API keys are all on your local device
- **No cloud database**: InkOS does not connect to any centralized database or cloud storage
- **No data transmission**: InkOS does not upload your data to any external server (unless you actively use third-party AI APIs)

## Reporting Security Vulnerabilities

### Process

1. **Do NOT publicly disclose**: Do not report security vulnerabilities in GitHub Issues, discussions, or other public channels
2. **Submit report**: Send via GitHub Security Advisories or open a private Issue

### Our Commitment

- **Response time**: Acknowledge receipt within 5 business days
- **Status updates**: Every 14 days during the fix process
- **Fix timeline**: Critical vulnerabilities fixed within 30 days of confirmation
